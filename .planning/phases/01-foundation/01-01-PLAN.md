<task type="auto">
  <name>Task 2: Configure PWA with vite-plugin-pwa</name>
  <files>vite.config.ts, public/manifest.json, src/main.tsx</files>
  <action>
    Install vite-plugin-pwa: `npm install -D vite-plugin-pwa`

    Configure in vite.config.ts:
    - Enable service worker with `registerType: 'autoUpdate'`
    - Set manifest with: name "Crossing Words", short_name "CrossingWords", theme_color "#1a1a2e", background_color "#1a1a2e" (dark theme default), display "standalone", start_url "/"
    - Add icons placeholder (192x192 and 512x512 — use placeholder SVGs for now, real icons come in Phase 8)

    Create placeholder icon files:
    - public/icon-192.svg (simple crossword grid icon as SVG)
    - public/icon-512.svg (same, larger)

    In src/main.tsx, add service worker registration check (vite-plugin-pwa handles this, just ensure no errors).

    Do NOT over-configure workbox caching — defaults are fine for development. Real caching strategy comes in Phase 8.
  </action>
  <verify>npm run build succeeds, dist/ contains manifest.json and sw.js, npm run preview shows installable PWA prompt in Chrome</verify>
  <done>PWA manifest configured, service worker generates on build, app is installable</done>
</task>
